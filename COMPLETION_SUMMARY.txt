## ✨ COMPLETE END-TO-END WORKFLOW

```
1. CUSTOMER VISITS SITE
   http://localhost:5173
   ↓
2. BROWSE MENU
   Click "Browse Menu" 
   → See categories (Pizza, Burgers, etc.)
   → Select category
   → See 3-10 products
   ↓
3. ADD TO CART
   Click product → Select size (if has sizes) → Add to cart
   (Repeats for multiple items)
   ↓
4. VIEW CART
   Click cart icon → See items, quantities, prices
   ↓
5. CHECKOUT
   Enter:
   • Name: "John Doe"
   • Phone: "03001234567"
   • Email: "john@example.com" (optional)
   • Address: "123 Main St" (if delivery)
   
   Select: Pickup OR Delivery (shows fee if delivery)
   ↓
6. PLACE ORDER
   Click "Place Order"
   ↓
   [BACKEND PROCESSING]
   • Validates customer info
   • Checks products exist
   • Calculates total server-side (secure)
   • Generates order number: ORD-{timestamp}-{random}
   • Creates WhatsApp message with order details
   ↓
7. ORDER CONFIRMATION
   Shows:
   • ✅ Order placed successfully!
   • Order number (copyable)
   • Total amount
   • "Open WhatsApp" button
   ↓
8. WHATSAPP MESSAGE
   Clicks "Open WhatsApp"
   → Pre-filled message sent to bakery
   → Message includes:
      - Customer name & phone
      - Order number
      - Items list with prices
      - Delivery address
      - Total amount
   ↓
9. TRACK ORDER
   Customer can go to "Track Order" page
   → Enter order number
   → See current status (Pending → Confirmed → Preparing → Delivered)
   → See items, address, total
   ↓
10. ADMIN MANAGEMENT
    Admin logs in to http://localhost:5173/admin
    → Sees all orders
    → Updates status (Pending → Confirmed → Preparing → Out for Delivery → Delivered)
    → Marks complete
    ↓
11. CUSTOMER RECEIVES NOTIFICATION
    Order status updates in real-time
    Customer can track delivery progress
```

## 🌐 API ENDPOINTS

All tested and working:

```
PUBLIC:
GET  /api/categories                        → List categories
GET  /api/products                          → List all products
GET  /api/products?category=Pizza           → Filter by category
GET  /api/products/:id                      → Single product
POST /api/orders                            → Create order (checkout)
GET  /api/orders/track/:orderNumber         → Track order

ADMIN (JWT required):
GET  /api/admin/orders                      → List all orders
GET  /api/admin/orders/:id                  → Order details
PATCH /api/admin/orders/:id/status          → Update status
```

## 📞 TROUBLESHOOTING

### "Menu not loading"
```bash
# Check backend is running
curl http://localhost:5000/api/products

# If fails, check .env has MongoDB URI and restart backend
# If works, frontend might have wrong API URL in .env
```

### "Checkout doesn't work"
```bash
# Open browser DevTools (F12) → Network tab
# Click "Place Order"
# Find POST /api/orders request
# Click it → Response tab → read error
# Common: "Cannot find product" → Check cart has correct product IDs
# Common: "Network Error" → Backend not running
```

### "Startup script doesn't work"
```bash
# Manual startup:
cd Backend && node seed.js && npm start
# (in new terminal)
cd Frontend && npm run dev
```
